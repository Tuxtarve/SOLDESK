write 주소 사용 환경변수 (호스트 셸에서 1회 설정)
export WRITE_API_BASE_URL="$(kubectl get svc write-api -n ticketing -o jsonpath='http://{.metadata.name}.{.metadata.namespace}.svc.cluster.local:{.spec.ports[0].port}')"

루트디렉토리 환경변수
export root_dir=

중요
- 호스트 셸에서 export 한 WRITE_API_BASE_URL 값은 kubectl exec ... sh -c '...' 내부로 자동 전달되지 않음
- 그래서 --via-was / --also-via-was 명령은 모두 sh -c 내부에서 export WRITE_API_BASE_URL='$WRITE_API_BASE_URL' 후 실행하도록 통일
- 즉, 호스트에서 export 1회 후 아래 명령 그대로 사용하면 됨


콘서트 실좌석. 기본=SQS 직접. --via-was=Write API만. --also-via-was=좌석 반씩 SQS+Write API.

kubectl -n ticketing exec -it tools-once -- sh -c 'cd /work/ticketing-db/terraform && python3 ../scripts/sqs_load_real_concert.py -n 100 --dry-run'

kubectl -n ticketing exec -it tools-once -- sh -c 'cd /work/ticketing-db/terraform && python3 ../scripts/sqs_load_real_concert.py -n 100'

kubectl -n ticketing exec -it tools-once -- sh -c "export WRITE_API_BASE_URL='$WRITE_API_BASE_URL' && cd /work/ticketing-db/terraform && python3 ../scripts/sqs_load_real_concert.py -n 3000 --spread-users 3000 --also-via-was"

kubectl -n ticketing exec -it tools-once -- sh -c "export WRITE_API_BASE_URL='$WRITE_API_BASE_URL' && cd /work/ticketing-db/terraform && python3 ../scripts/sqs_load_real_concert.py -n 20 --via-was"
